# 1.8 Backlog do Produto Ágil (User Stories)

> **O que é:** lista de funcionalidades que o software precisa ter, escrita
> sob a perspectiva de quem vai usar (visão Ágil).
>
> **Formato obrigatório:** *"Como [usuário], quero [funcionalidade] para [motivo/benefício]"*

---

## Definição de Pronto (DoD)

Uma User Story é considerada **Pronta** quando:

1. Código mergeado na branch `main`.
2. Rota acessível e funcional rodando localmente.
3. Sem erros no log do uvicorn.
4. Audit log gravado (quando aplicável).
5. Testada manualmente em Chrome + Firefox.

## Critérios de aceitação (exemplo detalhado — US-02)

> **US-02:** Como cliente, quero agendar um horário online…

| # | Critério | Como testar |
|---|---|---|
| AC-1 | Formulário aceita nome obrigatório | Submeter sem nome → mensagem de erro |
| AC-2 | Telefone e e-mail são opcionais | Submeter sem ambos → cadastro segue |
| AC-3 | Campo `consentimento_lgpd` é obrigatório | Desmarcar checkbox → bloqueio |
| AC-4 | Não permite agendar no passado | Selecionar data anterior → erro |
| AC-5 | Calendário Flatpickr aparece em tema dark | Abrir e conferir cor de fundo |
| AC-6 | Após sucesso, redireciona para `/agendar/ok` | Submissão válida exibe tela de confirmação |
